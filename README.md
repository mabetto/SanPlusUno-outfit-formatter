# HINASAKI Outfit Formatter

VRChat SDK3 アバター向けの衣装アニメーション生成ツールです。Modular Avatar と組み合わせて、衣装の切り替えアニメーター・メニュー・アイコンを一括生成します。

## 動作要件

- Unity 2022.3 LTS
- VRChat SDK3 (Avatars)
- [Modular Avatar](https://modular-avatar.nadena.dev/)

## インストール

Unity の `Window > Package Manager` を開き、`+` ボタン → **Add package from git URL** に以下を入力します。

```
https://github.com/mabetto/SanPlusUno-outfit-formatter.git
```

## ツールの起動

Unity メニューバーの **Tools > HINASAKI > OutfitFormatter** からウィンドウを開きます。

---

## 使い方

ウィンドウ上部の **簡単モード** / **詳細モード** タブでUIを切り替えられます。初めての場合は簡単モードを推奨します。

---

### 簡単モード

#### 1. 衣装プレハブ

ヒエラルキーまたは Project ウィンドウから衣装プレハブをフィールドにドロップします。

既存の MA メニュー設定が検出された場合、以下の3択が表示されます。

| ボタン | 動作 |
|---|---|
| メニューはそのまま（アニメーターのみ更新） | メニューに手を加えず、アニメーターと MA Parameters のみ更新します |
| 3+1形式に付け替える | 既存の MA Menu Installer を流用し、OutfitMenu にインストール先を切り替えます |
| メニューを上書き再生成 | メニューアセットを作り直して MA Menu Installer を上書きします |

#### 2. モードを選ぶ

| モード | 説明 |
|---|---|
| COS_Aポジション | 3+1 の OutfitMenu に追加し、COS_A パラメーターで制御します |
| COS_Bポジション | メニュー生成なし。アニメーターと MA Parameters のみ生成します |
| 独立 | 独自パラメーターで動作する独立した衣装として生成します |

**独立** を選んだ場合は「パラメーター名」と「インストール先メニュー」を追加で設定します。インストール先に **3+1 OutfitMenu** を指定すると、このツールが生成する OutfitMenu サブメニュー内に衣装コントロールを追加できます。

#### 3. メッシュを割り当てる

プレハブを設定すると、プレハブ内のレンダラー一覧が表示されます。

- チェックボックス（**使用する** / **スキップ**）で対象を選択します
- 各スロットにメッシュをドロップして割り当てます
- スロットごとに非表示条件を選択できます

| 非表示条件 | タイミング |
|---|---|
| COS_B時に非表示（キャストオフ時は表示） | CostumeBody=1（温泉タオル）時のみ非表示 |
| キャストオフ時に非表示 | CostumeBody=2（キャストオフ）時のみ非表示 |

**COS_Bポジション** モードの場合はプレハブ内の全レンダラー・PhysBone を自動で使用します。

#### 3a. COS_B 連携設定（COS_A ポジション共存時）

COS_A ポジションモードで、かつシーンに COS_B ポジションのオブジェクトが存在する場合に表示されます。COS_B との共存設定を行います。

#### 4. 出力設定

| 設定 | 説明 |
|---|---|
| 生成後にプレハブへ自動適用する | プレハブと同じフォルダに `プレハブ名_generated` フォルダを作成し、MA コンポーネントを自動でアタッチします |
| 出力フォルダ | 「選択」ボタンで出力先を変更できます（自動適用がオフの場合） |

#### アイコン設定

生成後にアイコンの差し替えや再生成が可能です。

- **既存アイコンを使用** にテクスチャをセットすると、そのアイコンをそのまま使用します（再レンダリング不要）
- **カメラ設定** でレンダリング角度・ズームを調整できます
  - **▶ シーンビューで位置・ズーム調整** を押すとシーンビュー上でハンドル操作ができます
  - 向きは **正面** / **背面** ボタンで切り替えられます
  - **Zoom** の値が小さいほど拡大されます
- **アイコンを再生成** ですでに生成済みのアイコンを再レンダリングします

#### 生成する

準備ができたら **生成する** ボタンを押します。

---

### 詳細モード

簡単モードの設定項目に加え、カテゴリ・パターン単位での細かい制御が可能です。

#### 1. プレハブ入力

簡単モードの「1. 衣装プレハブ」と同じです。

#### 2. モード選択

簡単モードの「2. モードを選ぶ」と同じです。

COS_A / COS_B ポジションの自動検出結果が **● 検出済み** / **○ 未検出** で表示されます。自動検出と実態が異なる場合のみ手動でオーバーライドしてください。

#### 3. カテゴリビルダー

衣装の切り替えパターンをカテゴリ・パターン単位で定義します。

- **メッシュ一覧を更新** でプレハブを再スキャンします
- **COS_Aテンプレート** でテンプレートを読み込みます

各カテゴリには以下を設定します。

| 設定 | 説明 |
|---|---|
| パラメーター | CostumeHead / CostumeBody / CostumeSkirt またはカスタム名（CostumeSkirt は SA_COS_A のパラメーターを再利用しているだけなので、スカート以外にジャケットやアクセサリーなど任意のパーツに割り当てても構いません） |
| 型 | Bool / Int |
| モード | Exclusive（排他）/ Combine（組み合わせ） |
| COS_B ポジションがある（value=2をCOS_B用に確保） | value=2 を COS_B 専用スロットとして固定します |

**+ パターン追加** でパターンを追加し、各パターンで制御するオブジェクトの **表示** / **非表示** と PhysBone の **有効** / **無効** を設定します。

#### 4. アイコン生成

簡単モードの「アイコン設定」と同じです。カテゴリ・パターンごとにアイコンを設定できます。

- **このパターンのアイコンを生成** で個別にレンダリングします

#### 5. 実行

| ボタン | 動作 |
|---|---|
| 適用方式（フォルダに出力 / Prefabに直接適用） | 出力方法を切り替えます |
| アニメーション生成 | AnimatorController を生成します |
| メニュー整形 | MA Menu Installer のメニューを整形します |
| lilToon明るさ同期のみ実行 | lilToon マテリアルの Brightness/Lighting(mono) レイヤーを同期します |
| すべて実行 | 上記をまとめて実行します |

---

## 注意事項

- **COS_Bポジション** モードは、アバターにすでに COS_B ポジションの衣装が存在する場合は使用できません（競合するため）。
- 初回実行後に設定は自動保存されます。次回起動時に自動で読み込まれます。
